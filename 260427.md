# 2026-04-27 작업 일지 (상단 탭 UI 가독성 고도화)

## 📌 주요 작업 내용

### 1. 상단 탭 잘림(Truncation) 문제 해결
- **요청 사항**: 기존 지정된 '우측 일렬 정렬(margin-right)' 구조에서, 카테고리의 총 넓이가 화면을 넘어갈 경우 왼쪽 탭들이 화면 바깥으로 밀려나가 텍스트가 잘리는(가독성 훼손) 문제 발생. 특히 모바일 화면과 긴 단어("EDUCATION & ACTIVITIES")에서 심각함.
- **해결 로직**:
  - **좌측 정렬로 패러다임 전환**: 인간의 기본 읽기 패턴(Left-to-Right)에 맞춰, 탭들을 `margin-left`를 이용한 0번 인덱스부터 좌측 정렬하는 형태로 변경 (`justify-content: flex-start`).
  - **동적 오버플로우(Overflow) 대응 알고리즘 적용**: `layoutFolders()` 함수 내에서 `containerWidth` 대비 탭들의 총합(`totalTabWidths`)을 계산. 필요한 간격이 화면보다 클 경우 `overlapGap`을 동적으로 축소(-값을 키움)시켜 화면 안에 무조건 딱 맞게 들어가도록 자동 렌더링.
  - **CSS 스케일 다운**: 좁은 모바일 화면을 대비하여 탭의 패딩(`0 18px` -> `0 6px`)과 폰트 사이즈 등을 함께 최적화시켜 텍스트 압축에 대응.

### 2. 가이드라인 및 규칙 지속 갱신
- **modul_story.md 갱신**: 제 9항 '상단 탭 우측 일렬 정렬' 규칙을 파기하고, 이를 대체하여 **'상단 탭 좌측 일렬 정렬 및 동적 스케일링(Horizontal Row Alignment & Dynamic Fits)'** 이라는 더욱 발전된 코어 규칙으로 덮어씀.
- **CLAUDE.md 갱신**: 해당 작업 내역 및 시간(26-04-27)을 핵심 규칙 테이블에 신규 등록하고, AI의 가독성 우선(Left Align + Fit) 행동 지침 재정의 완료.

### 3. 반응형 카테고리 탭 (Hover & Active) 오버레이 고도화
- **요청 사항**: 마우스를 올렸을 때 다른 카테고리에 가려져 있던 탭이 눈에 띄게 돌출되어 내용이 온전히 보이도록(아카이브/스튜디오 레퍼런스 스타일) 효과 적용.
- **처리 내용**:
  - 오버레이 (`.tab-overlay-click`) 시스템에 `mouseenter` 및 `mouseleave` 이벤트를 추가 연결하여, JS가 보이지 않는 호버를 감지하고 실제 `folder` 에 `.hovered` 클래스를 부여.
  - CSS에서 `.folder.hovered` 상태일 때의 `z-index`를 99(활성탭 바로 밑)로 끌어올림으로써, 가려져 있던 탭이 왼쪽의 다른 탭들 위로 즉시 돌출(Pop-up)되도록 렌더링.
  - 돌출된 탭의 시인성을 극대화하기 위해 `translateY(-6px)` 및 디자인적인 언더라인(밑줄) 효과를 부여하여 레퍼런스 이미지와 동일한 UX/UI 제공.

### 4. 경력(CAREER) 섹션 인더스트리얼 테마(Option C) 파싱 적용
- **요청 사항**: 데이터베이스에 입력된 장문의 경력(근무처, 카테고리, 상세업무) 텍스트를 구조화하여 시각적으로 쉽게 구분할 수 있는 C옵션 테마 적용.
- **처리 내용**:
  - `partials/03_career.html`에서 `c.company`를 분리하고 `c.role`의 렌더링 범위를 단독 문단으로 분리하여 헤드라인 가독성 침해 방지.
  - `formatBullets()` JS 로직을 대대적으로 리팩토링하여, `data.json`의 `[카테고리]` 내용을 인더스트리얼 테마의 라벨 탭(Badge) 컴포넌트로 자동 렌더링 (`.career-category-badge`).
  - `-` 불릿 형식의 상세 업무는 들여쓰기(Hanging Indent)와 세로선을 적용하여 전문적이고 문서화된 포맷으로 나열.
  - 회사명(`h3`) 텍스트에 Monospace 테마를 적용하여 캐비닛 서류 느낌을 배가.

### 5. 타임라인 지그재그 통일성(Option 1) 반영
- **요청 사항**: 왼쪽/오른쪽 교차 배치 시 왼쪽 항목이 강제 우측 정렬되어 가독성이 떨어지는 문제를 해결. 단, 기존에 정립된 둥근 아이콘, 가로/세로 간격, 대시 선 규칙 등은 **절대 훼손하지 않아야 함**.
- **처리 내용**:
  - `portfolio_server/templates/index.html` CSS 중 `.timeline-item:nth-child(odd)` 에 설정되었던 `text-align: right`를 `text-align: left`로 완전 통일 변경.
  - 관련하여 왼쪽 아이템의 세로 점선 보조선 방향도 우측(`padding-right`)에서 좌측(`padding-left`)으로 동일하게 정비하여 전후 양쪽이 거울처럼 완벽히 **Left-to-Right** 가독성을 유지하도록 조치.
