# 포트폴리오 디자인 및 시스템 불변 규정집 (modul_story.md)

이 문서는 사용자가 "현재 상태가 마음에 들어 더 이상 수정하지 않아도 된다"고 확정한 UI 디자인, 레이아웃, 그리고 시스템 로직을 모듈화하여 기록한 **영구 보존 룰북(Rulebook)**입니다. 
AI와 작업자가 다른 부분을 고도화할 때, **여기에 기록된 코어 모듈과 디자인은 절대로 훼손되거나 임의로 수정(오버라이드)되어서는 안 됩니다.**

### 1. 하단 폴더 탭 밀집(Deck) 레이아웃
- **확정 일시**: 2026-04-09 17:41
- **규정 대상**: `portfolio_server/templates/index.html` 내 JS `layoutFolders()` 함수
- **불변 원칙**:
    - 활성화된 탭(폴더) 하단에 있는 비활성 탭들은 **화면 맨 밑에서 밀착되어 쌓여야(Stack) 합니다.** (간격: `isMobile ? 12 : 16`)
    - 기존처럼 각 탭이 50px씩 공간을 차지하여 위쪽 본문(글 읽는 영역)을 좁게 만드는 에러가 발생해서는 절대 안 됩니다.
    - 파일 인덱스(카드 스와이프) 느낌을 강력히 유지합니다.

    - **핵심 색상**: 선은 다크그레이(`#2d3436`), 둥근 아이콘 테두리는 코랄/레드 포인트(`#ff6b6b`)를 유지합니다.
    - **구조**: 좌측으로 굵은 세로선(`::before`)이 뻗고, 각 항목마다 동그란 원형 이모지/이미지(`timeline-circle`)가 선 위에 위치하는 생산공정(Timeline) 다이어그램 구조를 지켜야 합니다.
    - 연도 배지는 진한 모노톤을 유지합니다.

### 6. 폴더 탭 클릭 오버레이(Tab Overlay) 시스템
- **확정 일시**: 2026-04-11 22:30
- **규정 대상**: `index.html` 내 `tab-overlay` 컨테이너 및 `setupTabDelegation()` 함수
- **불변 원칙**:
    - 활성 폴더의 본문(folder-body)이 다른 탭을 가려 클릭을 방해하는 문제를 해결하기 위해, 모든 탭 위에 투명한 클릭 전용 레이어를 생성하는 **오버레이 위임 방식**을 사용합니다.
    - `z-index: 9999`를 가진 `#tab-overlay`가 항상 최상단에 위치하며, JS가 실제 탭의 좌표를 계산하여 투명 버튼을 배치합니다. 
    - 이 로직은 `layoutFolders` 및 `window.resize` 시 `syncOverlayPositions()`를 통해 항상 동기화되어야 합니다.

### 7. 리스트 내어쓰기(Hanging Indent) 및 다단 구분선
- **확정 일시**: 2026-04-10 (고도화: 2026-04-11)
- **규정 대상**: `index.html` 하단 `<script>`의 `formatBullets()` 함수 및 CSS `.bullet-content`
- **불변 원칙**:

    - **고도화 사항**: 긴 문장이 줄바꿈되어 여러 줄이 될 때, 가독성을 위해 본문 영역 왼쪽에 **미세한 세로 구분선(`border-left: 2px solid rgba(0,0,0,0.08)`)**을 추가하여 문장의 범위를 명확히 구분합니다.

### 9. 상단 탭 좌측 일렬 정렬 및 동적 스케일링 (Horizontal Row Alignment & Dynamic Fits)
- **확정 일시**: 2026-04-27 (가독성 강화를 위해 좌우 정렬 구조 리팩토링)
- **규정 대상**: `portfolio_server/templates/index.html` 내 JS `layoutFolders()` 함수
- **불변 원칙**:
    - **가독성 최우선**: 사용자의 읽기 흐름(좌측->우측)을 고려해 **좌측에서부터 일렬로 나란히(가로선 상에서) 배치되도록** 해야 합니다.
    - **동적 공간 계산 (Overflow 방지)**: 만약 탭의 총 길이가 디바이스 화면을 초과하여 텍스트 짤림이 발생할 경우, JS에서 `availableWidth`를 구하고 `overlapGap`을 동적으로 늘려서(-값을 키워서) 반드시 한 화면 가로폭 안에 다 들어가게 하는 방어 코드가 고정 유지되어야 합니다.
    - 폴더 본체(`folder-body`)는 여전히 세로로 겹쳐서 내려가지만(`translateY`), 탭 자체는 역으로 밀려 올라가(`top: 2 - (i * cascadeGap)`) **시각적으로 같은 높이(Y축)** 에 위치해야 합니다.

### 10. 상단 탭 반응형 UX (Hover Pop-up & Underline)
- **확정 일시**: 2026-04-27
- **규정 대상**: `portfolio_server/templates/index.html` 내 JS `setupTabDelegation()` 함수 및 CSS `.hovered` 속성
- **불변 원칙**:
    - 탭들이 서로 겹쳐서 텍스트가 가려지는 상황을 타개하기 위해, 마우스 호버(`mouseenter`) 시 오버레이 클릭 영역이 감지하여 부모 `.folder`에 `.hovered` 클래스를 부여합니다.
    - 이 때 CSS로 해당 탭의 `z-index`를 최강제(99)로 끌어올려 가려진 부분을 시각적으로 뚫고 돌출(Pop-up)시켜 글자를 완전히 읽을 수 있도록 합니다.
    - 레퍼런스 디자인(`Archief`, `Studio` 래퍼런스)에 부합하도록 활성 및 호버된 탭에 검은색 밑줄(`text-decoration: underline`)을 적용해 시인성을 배가합니다.
