# 2026-04-14 작업 일지 (포트폴리오 UI 수정 및 GitHub 동기화)

## 📌 주요 작업 내용

### 1. 상단 탭 정렬 방식 변경 (지그재그 -> 일렬 나열)
- **요청 사항**: 기존의 지그재그식(계단식 가로/세로 이동) 탭 배열이 눈에 띄지 않는다는 피드백 반영.
- **수정 내용**: `portfolio_server/templates/index.html` 내 `layoutFolders()` 함수 수정.
  - 기존: `padding-left: ${(i / n) * 85}%` 및 `translateY(${i * cascadeGap}px)`를 사용하여 대각선 계단식 정렬.
  - 변경: 
    - 탭들을 세로(`Y축`)로는 모두 일직선상 상단에 위치하도록 `top: 2 - (i * cascadeGap)px` 를 부여하여 폴더 본체의 하강(translateY)을 상쇄.
    - 가로(`X축`)로는 각 탭의 실제 길이를 계산하여 우측 기준으로 차곡차곡 나열(`marginRight`).
    - 각 탭이 자연스럽게 -4px 정도 겹쳐 물리적인 포트폴리오 캐비닛 시각 효과 연출.
- **결과**: PC 및 Mobile 화면에서 명시적으로 탭들이 우측 상단에 정렬되어 존재감이 더욱 뚜렷해짐.

### 2. 정적 웹 변환 및 GitHub 배포 동기화
- **요청 사항**: `https://guardian-cabinet.github.io/`에 수동/서버 변경사항이 적용되지 않았음.
- **처리 내용**:
  - `render_portfolio.py` 스크립트를 수동으로 실행하여 `index.html` 정적 파일 업데이트.
  - Git을 연동하여 `git add`, `commit`, `push`를 터미널을 통해 실행함으로써 GitHub Pages가 업데이트되도록 트리거링.

### 3. 유지사항 및 룰북 기록 저장
- 해당 탭 변동 로직과 우측 정렬 룰은 앞으로의 UI 업데이트나 추가 렌더링 시에도 절대 훼손되지 않도록 `modul_story.md`의 **Rule 9** 및 추적(Tracking) 테이블에 명시적으로 추가 기록 완료.
- 말하지 않은 나머지 디자인(불릿 들여쓰기, 타임라인 오버레이, 색상 등)은 기존 Rulebook에 따라 안정적으로 호환 및 유지됨.
