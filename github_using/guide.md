# 📘 Portfolio Migration & Management Master Guide

본 문서는 **Render(Python)** 기반 포트폴리오에서 **GitHub Pages(Static)** 환경으로의 성공적인 이전 과정과, 향후 성능 고도화를 위한 **하네스 엔지니어링(Harness Engineering)** 루프 활용법을 정리한 마스터 가이드입니다.

---

## 1. 프로젝트 여정 요약 (Render → GitHub)

### [Phase 1] 기존 Render 환경의 한계
- **문제점:** 무료 플랜의 'Cold Start' 현상으로 인해 첫 접속 시 약 50초 이상의 로딩 지연 발생.
- **해결책:** 서버가 필요 없는 **정적 호스팅(GitHub Pages)** 방식으로 전환하여 즉각적인 로딩 속도 달성.

### [Phase 2] 정적 전환 및 보안 구현 (Current)
- **보안:** `security.js`를 통해 접속 시 ID/PIN을 요구하며, 인증 전에는 화면 노출을 차단함.
- **관리:** 개발 지식 없이도 `admin.html` 도구를 통해 로컬에서 내용을 수정하고 JSON 파일을 내려받는 방식으로 운영.
- **디자인:** 요청하신 '지그재그 타임라인'과 '블랙 프레임 헤더'를 오리지널 캐비닛 감성에 녹여내어 고유한 브랜드 정체성 확보.

## 2. 🔄 하네스 엔지니어링 (Harness Engineering) 루프
`bench/260401/` 폴더의 전문 파일들을 활용하여 포트폴리오의 성능과 디자인을 지속적으로 업그레이드하는 프로세스입니다.

1.  **[START] 계획 수립:** 개선이 필요한 영역(예: 모바일 가독성, 애니메이션)을 식별합니다.
2.  **[GENERATE] 코드 생성:** AI가 최신 트렌드를 반영한 고도화된 코드를 제안합니다.
3.  **[VALIDATE] 검증:** 생성된 화면을 실제 스크린샷과 디바이스에서 테스트합니다.
4.  **[EVALUATE] 채점:** 하네스 채점표(루브릭)에 따라 점수를 매깁니다. (90점 미만 시 반드시 수정 루프 진행)
5.  **[REVISE] 루프 반복:** 채점 피드백을 바탕으로 코드를 수정하여 성능을 극한으로 끌어올립니다.
    *   *현재 모바일 최적화 단계가 이 루프를 통해 완성되었습니다.*

## ⚠️ 반복된 실수 및 주의사항 (Precautions)

다음은 개발 과정 중 발생했던 주요 오류와 향후 작업 시 **반드시 주의해야 할 사항**입니다.

1.  **"지우지 말고 추가하라":** AI가 새로운 디자인(다크 테마 등)을 무리하게 제안하여 오리지널 감성을 잃은 적이 있습니다. **기존 스타일 유지**가 최우선 순위입니다.
2.  **코드의 완결성:** `admin.html`과 같은 중요한 도구에서 `...` (생략) 표시를 사용하는 것은 절대 금지입니다. 모든 빈칸과 기능은 100% 구현되어야 합니다.
3.  **한국어 가독성:** 한글 텍스트는 모바일에서 단어 중간이 잘리지 않도록 `word-break: keep-all;` 속성을 반드시 유지해야 합니다.
4.  **보안 유지:** 디자인을 아무리 고도화하더라도 `security.js` 연동이 끊어지면 안 됩니다.

## 🌐 GitHub Pages 배포 및 운영 워크플로우

사용자님께서 깃허브에 반영하실 때 다음 순서를 따라주세요.

### [Step 1] 관리자 도구에서 수정
1.  내 브라우저에서 `admin.html`을 엽니다.
2.  `content.json 가져오기` 버튼으로 현재 데이터를 불러와 수정합니다.
3.  **[💾 통합 저장 및 다운로드]**를 눌러 `config.json`과 `content.json`을 받습니다.

### [Step 2] 로컬 파일 교체
1.  내려받은 두 파일을 `C:\anti\homepage\github_using\data\` 폴더 안에 덮어씌웁니다.

### [Step 3] GitHub 업로드 (가장 중요)
현재 `github_using/` 폴더 내의 모든 파일이 최신 상태라면 다음과 같이 진행합니다:
1.  **GitHub Desktop** 또는 터미널을 사용합니다.
2.  `github_using/` 하위의 모든 파일들을 내 깃허브 레포지토리의 **Root(최상단)**에 복사합니다.
3.  변경 사항을 **Commit** 하고 **Push** 합니다.
4.  잠시 후 `https://[username].github.io/` 주소로 접속하면 전 세계 어디서든 업데이트된 포트폴리오를 볼 수 있습니다.
