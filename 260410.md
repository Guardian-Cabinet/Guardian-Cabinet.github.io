# 2026-04-10 업데이트 노트

## 완료된 작업 및 해결된 이슈

### 1. 카테고리(탭) 오클릭/영역 침범 버그 수정
- **증상:** 상단 카테고리 선택 시, 사용자의 마우스가 '01 PROFILE'에 있어도 다른 탭이 눌리는 오작동 발생.
- **원인:** CSS의 `.tab-wrapper` 태그가 100% 가로 폭으로 클릭 이벤트(`pointer-events: auto`)를 가로채어, 투명하게 겹쳐 있던 다른 폴더의 탭 영역이 클릭을 방해함.
- **해결:** 
  - `.tab-wrapper`의 클릭 이벤트를 비활성화(`pointer-events: none;`) 처리.
  - 실제 눈에 보이는 탭인 `.tab`에만 클릭 이벤트를 활성화(`pointer-events: auto;`)하여 정밀한 클릭이 가능하도록 개선.

### 2. 06번 'MILITARY' 글자 깨짐 및 시각적 오류 현상 방지
- **증상:** '06 AWARDS & MILITARY' 탭 라벨의 글자가 모바일/좁은 화면에서 깨지거나 줄바꿈되면서 레이아웃 붕괴 현상 발생.
- **해결:** 
  - `06_awards_military.html` 및 `index.html` 탭 CSS 룰에 `white-space: nowrap;` 추가.
  - 글자가 절대 두 줄로 깨지거나 잘리지 않고 한 줄로 강제 유지되도록 방어 코드를 작성하여 해결.

### 3. 긴 글의 뷸릿 포인트('-') Hanging Indent(내어쓰기) 자동 모듈화
- **증상:** 텍스트 내에서 `- 내용1...` 식으로 작성할 때, 내용이 길어져서 밑줄로 내려가면, 첫 줄의 `-` 문자 아래쪽까지 침범하여 가독성을 크게 훼방함.
- **원인:** 단순 `<p>` 태그에 텍스트만 쭉 늘어놓았기 때문. `<ul>, <li>`로 변환하지 않았음.
- **해결:** 
  - JavaScript 정규화 모듈(`formatBullets()`)을 `index.html` 하단에 새로 신설.
  - 웹을 띄우는 순간 자동으로 글을 순회하며 `- `(하이픈)으로 시작하는 문장들을 `div` 및 `flex-box` 레이아웃이 적용된 뷸릿 블록으로 자동 분리하여 감싸줌.
  - 이를 통해 `-` 문자는 좌측 고정이고 나머지 텍스트는 줄이 아무리 길게 개행되어도 `-` 라인 안쪽으로 깔끔하게 정렬(들여쓰기)되도록 완성.
